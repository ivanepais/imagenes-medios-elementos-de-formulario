Capitulo 9: Imágenes, medios y elementos de formularios

Estos elementos tienen un comportamiento diferente a otros en la web en cuanto se aplica CSS.

Elementos de reemplazo

Se denominan a las imagenes y videos, CSS no puede influir en su aspecto interno; sino solo a su posicion en la página y otros apartados. Tambien son otros elementos que tienen una relacion de aspecto: tamaño definido en ancho (x) y alto (y), por defecto se mostrarán en esta dimensión intrínseca.

Tamaño de las imágenes

  • Si tenemos una imágen más pequeña que la caja, la imagen no se va a expandir y quedara espacio en la caja.
  • Si tenemos una imágen más grande que la caja, la imágen desbordará

Para solucionar el desbordamiento podemos usar max-width: 100% en el primer escenario la imagen quedará pequeña y en el segundo, la imagen se reducirá. Funciona de igual forma para los elementos video y iframe.

Para solucionar el espacio en blanco podemos usar la propiedad object-fit con el valor cover tiene como caracteritica reducir la imagen para rellenar toda la caja manteniendo su relacion de aspecto (no se deforma). Si usamos el valor contain la imagen se reduce para mostrase completamente aunque dejando espacio en la caja. Si la caja es más pequeña que la imagen, esta queda con un aspecto panoramico. El valor fill rellena la caja pero no mantien su relacion de aspecto.

Elementos de reemplazo en una compaginación

En las tecnicas de diseño avanzadas podemos ver como se comportan las imagenes y los medios. Por ejemplo en un diseño flexible de cuadriculas, las imagenes se ubican al inicio de la pagina (arriba a la izquierda) en cambio los elementos se expanden hasta llenar el area.

Hasta ahora todos los ejemplos fueron de imagenes mas grande que la caja. Pero si la imagen es mas chica y queremos que se adapte por completo a la caja podemos usar widht y height: 100% o usar min-height: 100% pero perderemos definicion en la imagen. La mejor solución en ambos casos es que las imagenes y los medios tengan una resolucion adecuada para lo que necesite la web.

Elementos de formulario

Los formularios pueden ser complicados de si queremos diseñar con CSS. El articulo (link) da una explicacion más detallada

Los conceptos basicos son: input define campos de formulario de texto, selectores de hora y tambien de colores. El elemento textarea permite un espacio para texto en multilínea. Tambien hay elementos para contener y etiquetar partes de formularios como fieldset y legend

Html5 permite campos obligatorios o incluso para filtrar contenido a introducir por el usuario, así mismo mostrar un mensaje de error.

Estilo a elementos de entrada de texto

Como lo son input tipe="email" y textarea son faciles ya que se comportan como cajas. Sin embargo sus estilos prediterminados son diferentes en los OS y los navegadores.

Al aplicar estilos debemos dejar en claro que es un formulario. Los datos de entrada mas complejos se veran diferente en los OS siempre debemos suponer que los formularios se verán distintos y probarlos en distintos navegadores.

Herencia y elementos del formulario

En algunos navegadores, los elementos o heredan el estilo de letra por defecto. Si queremos que los formularios tengan nuestra fuente del body o de otro elemento principal, debemos agregar a los elementos del formulario: se establece al pricipio del CSS como regla general

  • button, input, select, textarea {}
  • font-family: inherit;
  • font-size: 100%;

Así heredará la fuente y su tamaño.

Elementos de formulario y tamaño de la caja

Como los formularios se presenta distinto en los navegadores, para una experiencia homogénea en cuanto a los anchos y alturas usamos box-sizing y los margenes y el padding lo dejamos en 0.

Se establece al princio del html como regla general de CSS y despues lo podemos modificar.

  • button, input, select, textarea {}
  • box-sizing: border box;
  • padding: 0;
  • margin: 0;

Ajustes útiles

En textarea se agrega overflow: auto; para que el navegador Internet Explorer no muestre la barra de desplazamiento cuando no hay necesidad.

Reinicio del formulario y conclusión

Para concluir, podemos poner estas propiedades juntas al comienzo al comienzo del CSS para reiniciar el formulario así los navegadores no influyen en el resultado final.

  • button, input, select, textarea {}
  • font-family: inherit;
  • font-size: 100%;
  • box-sizing: border-box;
  • padding: 0
  • margin: 0
  • textarea {}
  • overflow: 0;

Esta tecnica se llama normalize.css se trata de crear un conjunto de estilo base para los proyectos para que el sitio se vea de forma homogénea en los distintos navegadores del mercado.
